# Executive Summary

This project presents a structured cybersecurity risk quantification and control evaluation conducted on a simulated mobile banking platform (SecureMobile). The objective was to translate technical vulnerabilities into financial risk, evaluate control options, and provide data-driven recommendations for executive decision-making.

Using quantitative risk analysis methods, I calculated the Annualized Loss Expectancy (ALE) for three critical vulnerabilities: Database Injection, Session Hijacking, and API Authentication Bypass. The analysis revealed that Database Injection is the most significant risk, with an estimated annual financial exposure of $31,250,000. Session Hijacking and API Authentication Bypass follow with exposures of $3,000,000 and $750,000 respectively.

To mitigate these risks, I evaluated three control options: Web Application Firewall (WAF), Multi-Factor Authentication (MFA), and API Security Gateway. Each control was assessed based on cost, effectiveness, risk reduction, return on investment (ROI), and payback period.

The results show that the Web Application Firewall provides the highest financial benefit, reducing approximately $24,000,000 in risk exposure with a very high ROI. Multi-Factor Authentication also provides strong value, particularly in reducing account takeover risks, while the API Security Gateway offers additional protection with comparatively lower financial impact.

Based on the findings, I recommend a phased implementation strategy. The Web Application Firewall should be deployed first due to its high impact, followed by Multi-Factor Authentication as a quick and effective control. The API Security Gateway should be implemented as part of a long-term security improvement plan.

This project demonstrates how cybersecurity risk can be quantified and managed using financial metrics, enabling organizations to prioritize security investments based on measurable business impact.
